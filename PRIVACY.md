# 隐私说明

AetherX 是自托管软件，不提供承载用户业务数据的官方云服务。运行实例的个人或组织是该实例的数据管理者，应自行决定存储位置、网络暴露、备份和第三方 AI Provider。

## AetherX 处理的数据

根据使用的功能，Hub 可能保存：

- 账号名、显示名和密码哈希；
- 登录会话和配对设备令牌的哈希；
- 对话、长期记忆、偏好、画像、待办和提醒；
- 手记、成长事件、共同记忆、纪念册、梦境和图片；
- AI Provider 地址、模型名和加密后的 API Key；
- 多端同步变化游标和设备最后活动时间。

## 网络请求

客户端会连接用户选择的 AetherX Hub。使用 AI 功能时，Hub 会把完成请求所需的提示词、对话上下文和相关记忆发送到用户配置的 AI Provider；使用图像生成功能时，会向配置的图像 Provider 发送画面描述和必要参考内容。

具体数据处理还受所选 Provider 的条款和隐私政策约束。部署者应选择可信 Provider，并只发送完成任务所需的信息。

## 遥测

当前项目源代码没有内置产品分析、行为追踪或崩溃上报服务。第三方运行环境、代理、操作系统、AI Provider 或部署者自行增加的监控可能产生额外日志，这些不由 AetherX 项目控制。

## 凭证保护

- 密码只保存派生哈希；
- 会话和设备令牌只在数据库中保存哈希；
- AI API Key 使用 Hub 主密钥加密；
- 桌面端会话使用 Electron `safeStorage`；
- Android 长期令牌使用系统 Keystore；
- 手机端不保存 AI Provider Key。

这些措施不能替代安全的主机、HTTPS、访问控制和备份管理。

## 数据删除与备份

用户可以在产品界面中删除多类业务数据、注销会话或撤销设备。部署者仍需处理数据库备份中的历史副本。删除线上数据不会自动擦除既有备份；备份保留和销毁策略由实例部署者负责。

## 公开报告问题

提交 Issue 或 Pull Request 时，不要上传真实数据库、环境文件、对话、记忆、图片、API Key、密码或令牌。涉及漏洞时请按 [SECURITY.md](SECURITY.md) 私下报告。
