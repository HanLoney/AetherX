# ADR-0002：采用家庭节点单写与增量同步

- 状态：已接受
- 决策范围：桌面部署、手机连接、多端同步

## 背景

AetherX 需要让普通用户安装桌面程序后即可保存自己的数据，并允许手机端实时看到变化。项目以开源和自托管为前提，不建设承载用户数据的官方云服务。现有后端使用 SQLite，并已通过服务端会话进行用户隔离。

## 决策

每个安装实例运行一个 AetherX Hub。Hub 是该实例数据的唯一写入者，桌面端和手机端通过相同的业务 API 访问数据。

多端同步采用“REST 写入与读取、增量日志补拉、SSE 实时通知”的组合。手机端第一版不拥有独立的可写主数据库，不进行多主合并。

设备通过短时配对会话获得独立、可撤销的设备令牌。令牌映射到服务端账号，业务接口继续只信任认证上下文中的用户身份。

## 原因

- 复用现有 Node 后端、SQLite 数据和用户隔离；
- 保持单写，避免活跃 SQLite 文件跨设备复制；
- 增量游标同时解决实时通知和断线补拉；
- 手机端不接触 AI 密钥；
- 用户可以选择电脑、NAS 或私人服务器承载同一 Hub；
- 后续加入离线队列时不需要改写现有业务 API。

## 后果

- 电脑作为 Hub 时，电脑关闭后手机不能继续同步；
- 外网访问需要用户自己的 VPN、域名或服务器；
- 桌面安装包需要负责 Hub 生命周期、托盘和数据目录；
- 所有新增的用户业务表都必须进入增量日志覆盖范围；
- 手机离线写入和冲突处理延后到第二阶段。

## 暂不采用

- 官方托管数据库；
- 直接同步数据库文件；
- 默认公网暴露后端端口；
- 第一阶段引入 CRDT 或多主数据库。
