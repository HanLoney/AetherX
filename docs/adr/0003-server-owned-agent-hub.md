# ADR-0003：由后端统一托管 Agent Hub

- 状态：已采纳

## 背景

桌面端最初在渲染进程内完成提示词组合、模型调用、工具执行、授权和会话保存。
Android 客户端出现后，如果复制这套逻辑，两端会逐渐产生不同的 Markdown、工具
协议、空回复恢复和图片隐私行为，也难以保证同一账号数据的写入顺序。

## 决策

将完整 Agent Loop 放入 AetherX 后端：

- `POST /api/v1/agent/chat` 接收用户消息并返回服务端消息流；
- Hub 统一召回记忆并组合人格、心情和权威时间上下文；
- Hub 统一解析原生工具调用、旧版 `function_call` 与 DSML；
- 读工具直接执行，写入和删除工具暂停并返回 `approval_required`；
- 客户端通过 `POST /api/v1/agent/runs/{id}/approve` 继续或拒绝运行；
- Hub 统一保存展示消息和模型消息，并异步触发记忆提取与心情事件；
- 同一会话只允许一个活跃运行。

桌面端和手机端成为薄客户端，只保留输入、渲染、授权交互和增量同步。

## 结果

- 两端使用完全相同的提示词、工具、恢复与隐私规则；
- 手机无需依赖桌面窗口在线，只需能够访问同一个 Hub；
- Provider Key 仍只保存在 Hub；
- 待确认运行当前保存在内存中，Hub 重启或超时后不可继续，客户端需重新发起；
- 新增工具只注册一次，但桌面与手机需要理解相应的通用工具卡片数据结构。

## 未采用方案

- **桌面端作为转发中心**：要求桌面应用和窗口长期在线，移动端可用性较差；
- **两端各自实现 Agent Loop**：短期开发快，但协议、隐私和错误恢复必然分叉；
- **只同步最终文本**：无法可靠展示授权、工具状态和图片结果。
